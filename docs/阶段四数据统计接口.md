# 阶段四数据统计接口

## 数据库版本与部署状态

Flyway `V13__create_site_visit_events.sql` 新增 `site_visit_event`，保存官网有效访问事件：

- `SITE`：官网公共页面访问。
- `PROJECT`：婚礼项目详情成功加载。
- `COLLECTION`：作品集详情成功加载。
- 同一匿名访客对同一事件目标在 30 分钟内只计一次，避免刷新页面重复放大统计。
- 只保存访问日期、事件类型、内容编号、匿名访客 SHA-256 哈希、去重时间桶和创建时间。
- 不保存原始 IP、手机号、客户账号、访问密码或其他访客画像字段。

`V13` 已于 2026-07-18 完成生产部署，生产 schema 当前为 `V14`。生产字段
`visitor_hash` 为 `VARCHAR(64)`，与 Hibernate schema 校验保持一致。

## 官网访问采集

### `POST /api/public/analytics/visits`

公开接口，不要求登录。

官网页面访问：

```json
{
  "visitorId": "530b2cd5-8969-4c1d-ae2d-c0d459f67ed4",
  "type": "SITE",
  "targetId": null
}
```

项目或作品集浏览：

```json
{
  "visitorId": "530b2cd5-8969-4c1d-ae2d-c0d459f67ed4",
  "type": "PROJECT",
  "targetId": 12
}
```

成功接收返回 `204`。

规则：

- `visitorId` 由官网在浏览器本地生成并保存，只允许 16 至 100 位字母、数字、短横线和下划线。
- 服务端只保存 `visitorId` 的 SHA-256 哈希，不保存原始值。
- `SITE` 不允许携带 `targetId`。
- `PROJECT`、`COLLECTION` 必须携带正整数 `targetId`。
- 项目或作品集只有处于 `PUBLISHED` 且不是 `HIDDEN` 时才会计数。
- 无效或已经下架的内容编号不会落库，也不会通过响应暴露内容是否曾经存在。
- 相同访客、事件类型、目标和 30 分钟时间桶由数据库唯一约束兜底去重。

官网采集时机：

- 首页、项目列表、项目详情、作品集详情和客户评价公共路由进入后记录一次 `SITE`。
- 项目详情成功加载后记录 `PROJECT`。
- 作品集详情成功加载后记录 `COLLECTION`。
- 密码内容只有在正确解锁并成功加载详情后才记录内容浏览。
- 埋点请求不阻断页面展示，采集失败时官网继续正常使用。

## 管理员统计

### `GET /api/admin/analytics/overview`

要求管理员角色和 `/analytics` 权限。

参数：

- `days`：统计天数，允许 7 至 90，默认 30。

响应：

```json
{
  "days": 30,
  "startDate": "2026-06-19",
  "endDate": "2026-07-18",
  "summary": {
    "pageViews": 120,
    "uniqueVisitors": 48,
    "projectViews": 36,
    "collectionViews": 64,
    "inquiryCount": 5,
    "creatorUploadCount": 82,
    "pendingContent": 3,
    "rejectedContent": 1,
    "publishedContent": 12,
    "offlineContent": 2
  },
  "trend": [
    {
      "date": "2026-07-18",
      "pageViews": 12,
      "uniqueVisitors": 8,
      "projectViews": 4,
      "collectionViews": 7,
      "inquiryCount": 1,
      "creatorUploadCount": 10
    }
  ],
  "topProjects": [
    {
      "targetId": 12,
      "title": "湖畔婚礼",
      "views": 18,
      "uniqueVisitors": 12
    }
  ],
  "topCollections": []
}
```

统计口径：

- 页面、项目和作品浏览量均为经过 30 分钟去重后的有效访问事件数。
- 独立访客为统计周期内 `SITE` 事件的匿名访客哈希去重数。
- 咨询线索和创作者上传图片按所选周期统计。
- 待审核、部分驳回、已发布和已下架数量为查询时项目与作品集的当前状态合计。
- 趋势补齐周期内没有数据的日期，工作台可以稳定展示连续时间轴。
- 热门项目和作品集按有效浏览量倒序返回前 5 项，同时返回匿名访客数。

## 工作台入口

- 路由：`/console/analytics`
- 权限：`/analytics`
- 支持近 7 天、近 30 天切换和手动刷新。
- 展示访问概览、连续趋势、业务概览、热门婚礼项目和热门作品集。

## 当前边界

- 第一阶段不建设用户画像、访问来源、地域、设备、转化漏斗或跨设备访客合并。
- 匿名访客标识保存在浏览器本地；清理站点数据或更换浏览器后会形成新的匿名访客。
- 当前使用数据库事件明细和实时聚合；访问规模显著增长后再增加按日汇总表和历史数据归档。
- 后端完整测试、官网与工作台生产构建和正式域名桌面/移动端页面检查均已通过。
- 浏览器生产验收时将统计请求拦截为本地 `204`，没有为验收页面写入生产访问事件。
