# 阶段二图片接口

## 数据库版本

Flyway `V5__create_media_and_collection_photo_tables.sql` 新增：

- `media_asset`：原图、预览图、缩略图、文件摘要、真实尺寸和处理状态；高清品牌图路径由
  `original_path` 按固定目录规则派生，不新增数据库字段。
- `collection_photo`：作品集图片排序、审核状态和逻辑删除信息。
- `work_collection.cover_photo_id` 增加作品图片外键约束。

## 存储与安全规则

- 单张图片最大 30 MB，单次最多上传 50 张。
- Spring Multipart 请求总大小当前限制为 300 MB。
- 当前同步处理支持真实内容为 JPEG 或 PNG 的图片，不信任文件扩展名和请求 MIME。
- 图片最大边长为 20000 像素，总像素不超过 8000 万。
- 原图、高清品牌图、预览图和缩略图按 UTC 年月及 UUID 文件名保存。
- JPEG 会读取 EXIF Orientation，并在缩放前归一化全部 8 种旋转或镜像方向；原图文件不改写。
- 数据库保存 SHA-256、真实 MIME、转正后的展示宽高和原始文件大小。
- 预览图最长边不超过 1920 像素，并添加“糖诗·美学”透明品牌水印；水印位于右下角并按
  预览图尺寸缩放。
- 缩略图最长边不超过 480 像素。
- 高清品牌图最长边不超过 2560 像素，统一转为带“糖诗·美学”水印的 JPEG；原始 JPEG/PNG
  文件保持字节不变。
- Nginx 只公开 `/media/branded/`、`/media/previews/` 和 `/media/thumbnails/`；
  `/media/originals/` 固定返回 `404`。
- 后台图片管理接口仍只返回预览图和缩略图地址；公开官网接口只为已发布且允许访问的内容返回
  `originalUrl` 或 `coverOriginalUrl`。为兼容既有客户端保留字段名，但字段值指向高清品牌图，
  不再指向原始文件。
- 官网列表、封面、瀑布流和轮播使用缩略图或预览图；作品灯箱先显示预览图，再按需加载高清
  品牌图。
- 高清品牌图 URL 是可直接访问的静态地址；密码内容解锁后返回的地址可被持有者继续访问和
  分享，不具备逐次鉴权或防下载能力。
- 逻辑删除图片后数据库记录不可见，但四类物理文件长期保留。
- 生产发布会为历史原图补生成高清品牌图，不覆盖原始文件或既有预览图、缩略图。

## 权限与并发

以下接口要求权限资源 `/content/collections`：

- 管理员可以维护全部作品集图片。
- 创作者只能维护自己创建或参与的作品集图片。
- 已发布作品集不能上传、排序、设置封面或删除图片。
- 上传、实际排序变化、封面变化和删除会递增作品集 `version`；相同顺序或重复设置当前封面
  直接返回当前结果，不递增版本。
- 排序、封面和删除必须提交当前作品集 `version`。
- 图片上传、排序、封面和删除均记录操作日志。
- 上传事务回滚时会清理本次已生成但没有数据库记录的文件。

## 图片接口

### `GET /api/collections/{collectionId}/photos`

按 `sortOrder` 查询作品集全部未删除图片。该后台接口只包含预览图和缩略图公开地址，不返回
原图地址。

### `POST /api/collections/{collectionId}/photos`

使用 `multipart/form-data` 批量上传，字段名为 `files`。上传成功后图片状态为 `DRAFT`。

响应包含：

- `collectionVersion`：上传后最新作品集版本。
- `coverPhotoId`：当前封面图片编号。
- `photos`：完整的未删除图片列表。

### `PUT /api/collections/{collectionId}/photos/order`

请求字段：

- `version`：作品集当前版本。
- `photoIds`：全部未删除图片编号，按目标顺序排列，不允许缺失或重复。

顺序实际变化时，排序快照发生变化的图片回到 `DRAFT`，作品集必须重新提交并完成图片审核。

### `PUT /api/collections/{collectionId}/cover`

请求字段：

- `version`：作品集当前版本。
- `photoId`：该作品集中的未删除图片编号。

### `DELETE /api/collections/{collectionId}/photos/{photoId}?version={version}`

逻辑删除图片。删除当前封面时自动清空 `coverPhotoId`，物理文件不删除。
删除已通过图片属于审核内容变化，剩余有效图片会回到 `DRAFT` 并要求重新审核；删除驳回或
尚未通过的图片时，如果内容恢复为已审核基线，仍可按现有规则重新计算为 `READY`。

## 工作台入口

- 地址：`/console/collections/{collectionId}/photos`。
- 支持批量选择 JPEG/PNG、上传进度、预览、上移下移后完整保存排序、设置封面和逻辑删除。
- 存在未保存排序时，上传、封面和删除操作会先要求保存或放弃排序，避免覆盖本地顺序。
- 已发布作品集在页面和后端均锁定图片修改操作。

## 后续衔接

图片处理目前仍在上传请求内同步完成。图片提交审核、批量通过或驳回、删除驳回图片后的
状态重算、作品集发布下架流程见 `docs/阶段三审核发布与官网接口.md`。
