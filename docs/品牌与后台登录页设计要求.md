# 品牌与后台登录页设计要求

版本：V1.0
日期：2026-07-18
状态：已实施并于 2026-07-18 部署

## 1. 目标

将当前临时品牌 `Wedding Archive`、`Wedding Console` 和 `WA` 统一替换为正式品牌
“糖诗·美学”，并重做运营后台登录页，使官网的作品展示气质与后台的专业操作体验属于
同一个品牌系统。

本文件保留设计与验收要求，并同步记录当前实施结果。

## 2. 品牌基础

### 2.1 品牌名称

- 正式中文名：`糖诗·美学`。
- 必须保留中文间隔号 `·`，不使用普通句点、连字符或空格替代。
- 中文名是第一识别信息。
- 可选英文辅助名：`TANGSHI AESTHETICS`，只能作为小尺寸辅助字。

### 2.2 品牌含义

- “糖”表达婚礼中的温度、细节和愉悦感，但视觉上不做幼稚或过度甜腻处理。
- “诗”表达纪实影像、情绪和故事感。
- “美学”表达专业判断、作品质量和克制的编辑能力。

整体气质应是诗意、精致、温暖、可信赖和专业克制。官网可以更具画册与婚礼杂志感，
运营后台应保持安静、清晰和高效。

### 2.3 视觉方向

- 以炭黑、柔和白和深植物绿构成稳定基础，用少量莓果红作为状态或品牌强调色。
- 不使用大面积粉色、紫色渐变、金色高光、光斑或装饰性背景球体。
- 字体组合需要兼顾中文品牌气质和后台可读性，正文与表单不得使用装饰字体。
- Logo 不直接套用爱心、戒指、相机等常见素材图标，应从“糖诗”的字形、节奏或印章感中
  提炼具有识别度的标志。
- 页面中的照片必须展示真实婚礼项目或作品内容，不使用与业务无关的氛围素材。

## 3. Logo 资源要求

至少交付以下资源：

| 资源 | 用途 |
| --- | --- |
| 横版组合标 | 官网页头、工作台登录页和侧边栏 |
| 方形标志 | favicon、头像式标识和紧凑导航 |
| 中文标准字 | 首页首屏、页尾和品牌内容区 |
| 深色背景版本 | 照片或深色导航上的展示 |
| 浅色背景版本 | 白色和浅灰界面上的展示 |
| 单色版本 | 水印、打印和低干扰场景 |

源文件优先使用 SVG，位图导出至少提供透明 PNG 或 WebP。Logo 资源需要进入 Git，证书和
生产密钥仍按现有规则保留在 Git 之外。

Logo 在 16 px favicon、32 px 方形标志和手机端窄导航下仍需可辨识，不能依赖很细的线条或
只能在大尺寸下看清的文字细节。

## 4. 全站替换范围

实施时至少检查并替换：

- 官网首页、项目列表、项目详情和作品集详情的页头品牌。
- 官网首页首屏标题、页尾版权、浏览器标题、favicon 和无障碍名称。
- 工作台登录页、首次登录页、侧边导航和移动端导航中的品牌。
- 工作台与官网的 `index.html` 标题及 Vue Router 动态标题。
- 当前所有 `Wedding Archive`、`Wedding Console` 和 `WA` 用户可见文本。
- 公开预览图水印配置和后续生成的品牌水印资源。

英文辅助名不得成为首页 H1 或登录页唯一品牌名称。官网首页第一屏必须明确看到
“糖诗·美学”。

## 5. 图片水印

- 新生成的预览图使用“糖诗·美学”单色 Logo 或组合水印，保留必要的域名识别信息。
- 建议使用透明位图 Logo 合成水印，避免依赖 iot 服务器是否安装指定中文字体。
- 水印保持低干扰，不能遮挡人物面部和作品关键内容。
- 水印预览图继续供工作台审核和预览使用；官网业务图片按后续要求直接使用原图。
- 仅修改水印配置不会影响既有预览图，上线前必须决定是否批量重建历史预览图。
- 如果重建历史预览图，需要提供可重复执行、失败可恢复和不修改原图的处理脚本，并在生产执行前
  备份现有预览图目录。

本次实施只对后续新上传图片生成品牌水印预览图，没有批量重建历史预览图。这样可以避免在
没有独立备份、执行日志和失败恢复脚本时覆盖线上已有派生文件。

## 6. 运营后台登录页

### 6.1 页面结构

- 登录页第一视觉必须是“糖诗·美学”Logo 和中文品牌名。
- 使用真实且清晰的婚礼作品作为视觉背景或主要画面，不使用模糊、过暗或纯装饰图片。
- 登录表单是清晰独立的操作区域，卡片圆角不超过 8 px，不做营销落地页式功能介绍。
- 桌面端减少当前大面积空白，品牌画面与登录表单保持明确层级。
- 手机端优先保证表单完整可见，键盘弹出后登录按钮和错误信息仍可访问。

### 6.2 表单与交互

- 保留手机号和密码登录，不增加验证码。
- 手机号输入使用数字键盘和正确的自动填充语义。
- 密码框提供熟悉的显隐图标按钮，并带可访问名称和提示。
- 登录按钮具有默认、悬停、按下、加载和禁用状态，加载期间阻止重复提交。
- `401` 明确提示手机号或密码错误；网络、服务和权限错误使用不同的可理解提示。
- 支持 Enter 提交、Tab 顺序和清晰焦点状态。
- 不保存密码，可按后续确认决定是否只记住上次手机号。
- 登录成功后继续复用现有首次改密和资料完善流程。

### 6.3 响应式与视觉验收

- 至少检查 `1440x1000`、`1024x768`、`390x844` 和 `360x800` 视口。
- 不允许页面横向滚动、文字裁切、表单溢出或键盘遮挡关键操作。
- Logo、标题、字段、错误信息和按钮之间不得重叠。
- 图片加载失败时保留可用的纯色背景和完整登录表单。
- 登录页首屏不得依赖滚动才能看到登录按钮。
- 正文、占位符、错误和禁用状态满足基本对比度要求。

## 7. 实施约束

- 先形成 Logo 资源和设计令牌，再替换页面，避免各页面独立拼接品牌文本。
- 官网与工作台共享同一品牌资产源，允许根据深浅背景选择不同版本。
- 不改变现有认证接口、JWT、CORS 和首次登录业务规则。
- 不在本轮引入后台动态修改品牌配置，品牌仍通过版本化静态资源发布。
- 前端构建完成后必须执行桌面和手机截图检查；涉及图片背景时确认资源真实加载且没有裁切主体。
- 品牌水印实施需要补后端测试，验证新上传图片生成预览图且原图文件保持不变。

## 8. 验收清单

1. 官网和工作台不再出现旧临时品牌文本。
2. 官网第一屏、后台登录页和侧边栏均清晰展示“糖诗·美学”。
3. favicon、浏览器标题、无障碍名称和页尾同步更新。
4. Logo 在深色、浅色、桌面和手机场景下均可辨识。
5. 登录表单的密码显隐、加载、错误、键盘和响应式状态完整可用。
6. 登录成功、失败和首次改密流程未发生认证回归。
7. 新上传预览图使用品牌水印，官网业务图片通过公开原图 URL 展示。
8. 如处理历史预览图，迁移过程有备份、日志和失败恢复方式。
9. 官网和工作台生产构建通过，目标视口无横向溢出、重叠或裁切。
10. 实施完成后同步更新 `docs/开发交接.md` 和 `CHANGELOG.md`，记录实际验证和部署状态。

## 9. 实施结果

- 根品牌资产位于 `brand/`，包含横版组合标、方形标志、深浅背景、单色、favicon、SVG 水印
  和透明 PNG 水印；`deploy/scripts/sync-brand-assets.sh` 负责同步运行时副本。
- 官网首页、项目列表、项目详情、作品集详情、页尾、浏览器标题、favicon 和无障碍名称已替换。
- 工作台登录页、首次登录页、侧边栏、移动导航、浏览器标题和 favicon 已替换。
- 登录页使用真实婚礼照片，手机号密码认证流程保持不变，并增加密码显隐、加载防重和错误分级。
- 新上传图片的预览图使用透明品牌水印；缩略图、原图和历史预览图保持不变。
- 后端测试、两个前端生产构建、脚本语法、桌面和手机视觉检查均已执行；实际命令和部署结果见
  `docs/开发交接.md` 与根目录 `CHANGELOG.md`。
- 生产部署备份目录为 `/home/apps/wedding-platform/backups/20260718-103325`，没有新增
  Flyway 迁移，生产 schema 保持 `V7`。
