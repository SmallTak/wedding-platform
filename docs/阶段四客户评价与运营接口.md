# 阶段四客户评价与运营接口

## 数据库版本与部署状态

Flyway `V8__create_feedback_operations_tables.sql` 新增：

- `customer_feedback`：客户评价、被评价项目与创作者、星级、审核和公开状态。
- `feedback_reply`：创作者对单条评价的唯一回复及独立审核状态。
- `consultation_lead`：官网咨询内容、来源、跟进状态、跟进管理员和乐观锁版本。
- `homepage_feature`：首页项目、作品集和评价的推荐、置顶、顺序及逻辑停用记录。
- 权限资源 `/operations/feedback`，分配给管理员和创作者。

`/operations/inquiries` 和 `/site/home` 已由 `V2` 建立，并由 `V3` 分配给管理员。

`V8` 已在独立 H2 环境完成 `V1` 至 `V8` 迁移和接口流程验证，截至 2026-07-18
尚未部署到生产环境；生产 schema 仍为 `V7`。

## 客户评价

### 权限和数据范围

工作台评价接口统一要求权限资源 `/operations/feedback`：

- 管理员可以查看全部评价，并为任意项目参与创作者代提交评价。
- 创作者只查看与自己相关的评价，只能在自己创建或参与的项目中为自己代提交评价。
- 被评价创作者必须是所选项目的创建人或参与者。
- 修改和撤回只允许管理员或原提交人执行。
- 当前没有客户注册、项目关联和客户中心，`customer_user_id` 暂不由现有流程写入。

### 状态规则

评价审核状态：

- `PENDING`：待管理员审核。
- `APPROVED`：已通过。
- `REJECTED`：已驳回，可由原提交人或管理员修改后重新提交。

评价公开状态：

- `UNPUBLISHED`：尚未公开，可以修改或撤回。
- `PUBLISHED`：已公开并锁定，只能由管理员下架。
- `OFFLINE`：已下架并继续锁定，不能修改或撤回。

管理员通过评价时，关联项目必须为 `PUBLISHED + PUBLIC`；通过操作会同时将评价公开。
项目下架、隐藏或删除后，评价即使仍为 `PUBLISHED` 也不会出现在公开接口。

### 工作台接口

#### `GET /api/feedback`

分页查询评价，支持：

- `page`、`size`，每页最多 100 条。
- `reviewStatus`
- `publishStatus`
- `projectId`

管理员返回全部数据，创作者只返回被评价创作者为自己的数据。

#### `GET /api/feedback/options`

返回当前账号可选择的项目和创作者：

- 管理员返回全部未删除项目和有效创作者。
- 创作者返回自己可访问的项目，并且只能选择自己。

#### `POST /api/feedback`

管理员或创作者代提交评价：

```json
{
  "projectId": 12,
  "creatorUserId": 8,
  "customerDisplayName": "林女士",
  "rating": 5,
  "content": "记录了很多真诚瞬间"
}
```

客户称呼最多 100 个字符，星级为 1 至 5，评价内容最多 2000 个字符。
创建后状态为 `PENDING + UNPUBLISHED`。

#### `PUT /api/feedback/{feedbackId}`

请求结构与创建相同，并增加当前 `version`。只允许管理员或原提交人修改尚未公开的评价。
保存后清除旧审核结论并回到 `PENDING + UNPUBLISHED`。

#### `DELETE /api/feedback/{feedbackId}?version={version}`

审核公开前撤回评价，成功返回 `204`。评价和已有回复统一逻辑删除，物理记录和审计日志保留。
已公开或已下架评价拒绝撤回。

#### `PUT /api/feedback/{feedbackId}/reply`

只有被评价创作者本人可以回复已经公开的评价：

```json
{
  "version": null,
  "content": "谢谢你们把信任交给我们"
}
```

首次回复时 `version` 为空；修改已驳回或待审回复时提交回复当前版本。每条评价最多一条回复，
回复提交后为 `PENDING`。已通过回复保持锁定。

### 管理员审核接口

以下接口要求管理员角色和 `/operations/feedback` 权限：

- `POST /api/admin/feedback/{feedbackId}/approve`
- `POST /api/admin/feedback/{feedbackId}/reject`
- `POST /api/admin/feedback/{feedbackId}/offline`
- `POST /api/admin/feedback/{feedbackId}/reply/approve`
- `POST /api/admin/feedback/{feedbackId}/reply/reject`

通过请求只包含 `version`：

```json
{
  "version": 2
}
```

评价或回复驳回请求包含 `version` 和最多 500 个字符的必填 `reason`。评价下架请求结构相同；
只有 `PUBLISHED` 评价可以下架。全部修改、撤回、审核、公开、回复和下架操作写入操作日志。

### 官网公开接口

#### `GET /api/public/feedback`

支持 `page` 和 `size`，每页最多 60 条。只返回满足以下条件的评价：

- 评价为 `APPROVED + PUBLISHED`。
- 关联项目为 `PUBLISHED + PUBLIC` 且未删除。

客户称呼统一显示首字符加两个星号，空称呼回退为“匿名客户”。只有 `APPROVED` 且已发布的
创作者回复会随评价公开。

公开项目详情 `GET /api/public/projects/{projectId}` 同时返回该项目下的公开评价。
官网页面入口为 `/reviews`，项目详情页也显示当前项目评价。

## 咨询线索

### `POST /api/public/inquiries`

无需登录，成功返回 `201`：

```json
{
  "name": "周女士",
  "contact": "微信 tangshi-customer",
  "weddingDate": "2026-10-18",
  "region": "杭州",
  "serviceNeeds": "婚礼摄影与跟拍",
  "remark": "希望了解全天服务",
  "website": ""
}
```

`name`、`contact` 和 `serviceNeeds` 必填。`website` 是隐藏蜜罐字段，非空时返回通用接收结果但
不创建线索。真实请求按客户端 IP 在单进程内限制为每小时最多 5 次；生产 Nginx 会将
`X-Real-IP` 和 `X-Forwarded-For` 覆盖为直接客户端地址，后端优先使用可信 `X-Real-IP`，
不接受外部请求伪造的首段转发地址。过期限流桶会周期清理，应用重启后内存计数清空。
创建成功后返回以 `INQ` 开头的唯一咨询编号，初始状态为 `NEW`，来源为 `WEBSITE`。

### 管理员接口

以下接口要求管理员角色和 `/operations/inquiries` 权限：

- `GET /api/admin/inquiries`
- `PUT /api/admin/inquiries/{leadId}`

列表支持 `page`、`size`、`keyword` 和 `followStatus`，每页最多 100 条。关键词匹配咨询编号、
姓名、联系方式、地区和服务需求。

跟进状态包括：

- `NEW`
- `CONTACTED`
- `FOLLOWING`
- `COMPLETED`
- `INVALID`

更新请求：

```json
{
  "version": 0,
  "followStatus": "CONTACTED",
  "followNote": "已通过微信联系"
}
```

保存时自动将当前管理员记为 `assignedAdminId`，并写入操作日志。

## 首页人工推荐

首页推荐支持三种目标：

- `PROJECT`
- `COLLECTION`
- `FEEDBACK`

只有当前公开发布的内容可以进入推荐候选和保存结果。管理员接口要求管理员角色和
`/site/home` 权限。

### `GET /api/admin/site/home`

返回当前可推荐的公开项目、公开作品集、公开评价和有效推荐配置。已下架、隐藏、删除或失效的
推荐记录不会出现在管理选项中。

### `PUT /api/admin/site/home`

全量替换首页推荐：

```json
{
  "items": [
    {
      "targetType": "PROJECT",
      "targetId": 12,
      "sortOrder": 10,
      "pinned": true
    },
    {
      "targetType": "FEEDBACK",
      "targetId": 7,
      "sortOrder": 20,
      "pinned": false
    }
  ]
}
```

同一类型和目标不能重复。项目最多 6 个、作品集最多 12 个、评价最多 6 条。排序为置顶优先，
再按 `sortOrder` 和记录编号。替换操作将旧配置逻辑停用；重新选择同一目标时复用并激活原记录，
不物理删除历史配置。

### `GET /api/public/home`

无需登录，返回首页项目、作品集和评价：

- 有有效人工配置时，按配置顺序返回当前仍公开的内容，失效记录不占展示数量。
- 某一类型没有有效人工配置时，项目回退最近发布 3 个，作品集回退最近发布 6 个，
  评价回退最近发布 3 条。

官网首页已经接入该接口，同时提供客户评价区和咨询提交表单。工作台页面入口为
`/console/site-home`，咨询线索为 `/console/inquiries`，客户评价为 `/console/feedback`。

## 当前边界

- 客户注册、登录、项目关联申请、客户个人中心和客户直接提交评价尚未实现。
- 当前评价由管理员或创作者代提交；数据库已预留可空 `customer_user_id`。
- 咨询频率限制保存在单个应用进程内，不适用于多实例共享计数。
- 首页主题、宣传语、品牌主色和联系方式仍通过版本化前端资源发布，不在本接口动态配置。
- 官网访问量、项目浏览量、作品集浏览量和趋势统计尚未实现。
