## 客户评价

### 权限和数据范围

工作台评价接口统一要求权限资源 `/operations/feedback`：

- 管理员可以查看全部评价，并为任意作品集的创建者或共同创作者代提交评价。
- 当前工作台评价提交、修改、审核和回复维护仅允许管理员操作。
- 被评价创作者必须是所选作品集的创建者或共同创作者。
- 修改和撤回只允许管理员或原提交人执行。
- 客户项目关联与客户本人直接提交评价已随 V15 下线；客户中心仅保留历史消息、我的评价和账号设置。

### 状态规则

评价审核状态：

- `PENDING`：待管理员审核。
- `APPROVED`：已通过。
- `REJECTED`：已驳回，可由原提交人或管理员修改后重新提交。

评价公开状态：

- `UNPUBLISHED`：尚未公开，可以修改或撤回。
- `PUBLISHED`：已公开并锁定，只能由管理员下架。
- `OFFLINE`：已下架并继续锁定，不能修改或撤回。

管理员通过评价时，关联作品集必须为 `PUBLISHED + PUBLIC`；通过操作会同时将评价公开。
作品集下架、隐藏或删除后，评价即使仍为 `PUBLISHED` 也不会出现在公开接口。

### 工作台接口

#### `GET /api/feedback`

分页查询评价，支持：

- `page`、`size`，每页最多 100 条。
- `reviewStatus`
- `publishStatus`
- `collectionId`

管理员返回全部数据。

#### `GET /api/feedback/options`

返回当前账号可选择的作品集和创作者：

- 管理员返回全部未删除作品集和有效创作者。
- 创作者不再提交评价，工作台入口由权限控制隐藏。

#### `POST /api/feedback`

管理员代提交评价：

```json
{
  "collectionId": 12,
  "creatorUserId": 8,
  "customerDisplayName": "林女士",
  "rating": 5,
  "content": "记录了很多真诚瞬间"
}
```

客户称呼最多 100 个字符，星级为 1 至 5，评价内容最多 2000 个字符。
创建后状态为 `PENDING + UNPUBLISHED`。

#### `PUT /api/feedback/{feedbackId}`

请求结构与创建相同，并增加当前 `version`。只允许管理员或原提交人修改尚未公开的评价。
保存后清除旧审核结论并回到 `PENDING + UNPUBLISHED`。

#### `DELETE /api/feedback/{feedbackId}?version={version}`

审核公开前撤回评价，成功返回 `204`。评价和已有回复统一逻辑删除，物理记录和审计日志保留。
已公开或已下架评价拒绝撤回。

#### `PUT /api/feedback/{feedbackId}/reply`

管理员可以为已经公开的评价维护回复：

```json
{
  "version": null,
  "content": "谢谢你们把信任交给我们"
}
```

首次回复时 `version` 为空；修改已驳回或待审回复时提交回复当前版本。每条评价最多一条回复，
回复提交后为 `PENDING`。已通过回复保持锁定。

### 管理员审核接口

以下接口要求管理员角色和 `/operations/feedback` 权限：

- `POST /api/admin/feedback/{feedbackId}/approve`
- `POST /api/admin/feedback/{feedbackId}/reject`
- `POST /api/admin/feedback/{feedbackId}/offline`
- `POST /api/admin/feedback/{feedbackId}/reply/approve`
- `POST /api/admin/feedback/{feedbackId}/reply/reject`

通过请求只包含 `version`：

```json
{
  "version": 2
}
```

评价或回复驳回请求包含 `version` 和最多 500 个字符的必填 `reason`。评价下架请求结构相同；
只有 `PUBLISHED` 评价可以下架。全部修改、撤回、审核、公开、回复和下架操作写入操作日志。

### 官网公开接口

#### `GET /api/public/feedback`

支持 `page` 和 `size`，每页最多 60 条。只返回满足以下条件的评价：

- 评价为 `APPROVED + PUBLISHED`。
- 关联作品集为 `PUBLISHED + PUBLIC` 且未删除。

客户称呼统一显示首字符加两个星号，空称呼回退为“匿名客户”。只有 `APPROVED` 且已发布的
创作者回复会随评价公开。

官网页面入口为 `/reviews`；作品集相关公开评价可由作品集维度筛选展示。

## 咨询线索

### `POST /api/public/inquiries`

无需登录，成功返回 `201`：

```json
{
  "name": "周女士",
  "contact": "微信 tangshi-customer",
  "weddingDate": "2026-10-18",
  "region": "杭州",
  "serviceNeeds": "婚礼摄影与跟拍",
  "remark": "希望了解全天服务",
  "website": ""
}
```

`name`、`contact` 和 `serviceNeeds` 必填。`website` 是隐藏蜜罐字段，非空时返回通用接收结果但
不创建线索。真实请求按客户端 IP 在单进程内限制为每小时最多 5 次；生产 Nginx 会将
`X-Real-IP` 和 `X-Forwarded-For` 覆盖为直接客户端地址，后端优先使用可信 `X-Real-IP`，
不接受外部请求伪造的首段转发地址。过期限流桶会周期清理，应用重启后内存计数清空。
创建成功后返回以 `INQ` 开头的唯一咨询编号，初始状态为 `NEW`，来源为 `WEBSITE`。

### 管理员接口

以下接口要求管理员角色和 `/operations/inquiries` 权限：

- `GET /api/admin/inquiries`
- `PUT /api/admin/inquiries/{leadId}`

列表支持 `page`、`size`、`keyword` 和 `followStatus`，每页最多 100 条。关键词匹配咨询编号、
姓名、联系方式、地区和服务需求。

跟进状态包括：

- `NEW`
- `CONTACTED`
- `FOLLOWING`
- `COMPLETED`
- `INVALID`

更新请求：

```json
{
  "version": 0,
  "followStatus": "CONTACTED",
  "followNote": "已通过微信联系"
}
```

保存时自动将当前管理员记为 `assignedAdminId`，并写入操作日志。

## 首页人工推荐

首页人工推荐当前仅支持客户评价目标：

- `FEEDBACK`

只有当前公开发布的内容可以进入推荐候选和保存结果。管理员接口要求管理员角色和
`/site/home` 权限。

### `GET /api/admin/site/home`

返回当前可推荐的公开评价和有效推荐配置。历史项目/作品集推荐已由 V15 清理。已下架、隐藏、
删除或失效的推荐记录不会出现在管理选项中。

### `PUT /api/admin/site/home`

全量替换首页推荐：

```json
{
  "items": [
    {
      "targetType": "FEEDBACK",
      "targetId": 7,
      "sortOrder": 20,
      "pinned": false
    }
  ]
}
```

同一类型和目标不能重复。评价最多 6 条；`PROJECT` 和 `COLLECTION` 不再接受。排序为置顶
优先，再按 `sortOrder` 和记录编号。替换操作将旧配置逻辑停用；重新选择同一目标时复用并
激活原记录，不物理删除历史配置。

### `GET /api/public/home`

无需登录，返回兼容用作品集列表、评价和作品轮播：

- 有有效评价人工配置时，按配置顺序返回当前仍公开的评价，失效记录不占展示数量。
- 没有有效评价人工配置时，评价回退最近发布 3 条。
- `collections` 继续返回最近发布 6 个公开作品集，用于首页轮播兜底和旧客户端兼容；新版首页
  默认作品区优先显示 `carousel`，搜索和分类结果继续调用公开作品列表接口。

官网首页已经接入该接口，同时提供客户评价区和咨询提交表单。工作台页面入口为
`/console/site-home`，咨询线索为 `/console/inquiries`，客户评价为 `/console/feedback`。

## 首页作品轮播

首页轮播以作品照片 `photoId` 为配置目标。候选必须同时满足：

- 作品集审核状态为 `APPROVED`。
- 作品集为 `PUBLISHED + PUBLIC` 且未删除。
- 候选照片属于当前作品集且图片审核状态为 `APPROVED`。
- 照片媒体未删除、处理状态为 `SUCCESS`，并存在原图、预览图和缩略图路径；公开响应中的
  `originalUrl` 使用受控图片 API。

后台配置接口要求管理员角色和 `/site/home` 权限。

### `GET /api/admin/site/home/carousel`

返回当前可选作品照片和已配置轮播项：

```json
      "description": "湖风、草坪和傍晚的仪式",
```

- `COLLECTION_COVER_REQUIRED`

### `PUT /api/admin/site/home/carousel`

全量替换轮播配置，请求最多包含 5 项；同一 `photoId` 不得重复：

```json
      "sortOrder": 10,

      "focalY": 50.00
    }
  ]
}
```

`focalX` 和 `focalY` 范围均为 `0` 至 `100`，分别表示图片水平和垂直焦点百分比。保存前服务端
重新校验照片和所属作品集仍满足候选条件；替换旧配置采用逻辑停用和重新激活，并写入
`REPLACE_HOMEPAGE_CAROUSEL` 操作日志。
