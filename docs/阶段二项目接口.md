# 阶段二项目接口

## 数据库版本

Flyway `V4__create_project_and_collection_tables.sql` 新增：

- `wedding_project`：婚礼项目及独立的审核、发布、可见性状态。
- `project_creator`：项目参与创作者和分配信息。
- `content_category`、`content_tag`：作品分类与标签。
- `work_collection`：项目内或独立作品集。
- `collection_creator`、`collection_tag`：作品集创作者与标签关系。

项目、分类、标签和作品集统一包含创建人、更新人、时间、乐观锁版本和逻辑删除字段。
项目封面接口尚未实现，`cover_asset_id` 暂不设置外键；作品集封面由 Flyway `V5`
通过 `cover_photo_id` 关联作品图片。

## 权限与数据范围

以下接口均要求 JWT 中包含权限资源 `/content/projects`：

- 管理员可以查询和维护全部项目。
- 创作者只能查询和维护自己创建或已被分配参与的项目。
- 创作者创建项目后自动成为参与者，管理员分配参与者时不能移除项目创建者。
- 只有管理员角色可以分配项目参与者。
- 更新项目或参与者时必须提交最新 `version`，版本过期返回 `409`。
- 已发布项目的公共资料不可编辑；阶段三提供下架和重新提交审核流程。

## 项目接口

### `POST /api/projects`

创建婚礼项目。请求字段：

- `title`：必填，最长 200 字符。
- `coupleDisplayName`：新人脱敏展示名称，可空。
- `eventDate`：必填，格式 `yyyy-MM-dd`。
- `regionCode`：必填，地区编码。
- `locationText`：必填，地点文本。
- `description`：可空，最长 5000 字符。

项目编号由系统生成。新项目固定初始化为：

- `visibility=HIDDEN`
- `reviewStatus=DRAFT`
- `publishStatus=UNPUBLISHED`

### `GET /api/projects`

分页查询当前账号可访问的项目。

查询参数：

- `page`：从 `0` 开始，默认 `0`。
- `size`：默认 `20`，范围 `1` 至 `100`。
- `keyword`：可选，匹配项目编号、标题、新人展示名称和地点。

### `GET /api/projects/{projectId}`

查询项目详情及当前参与创作者。

### `PUT /api/projects/{projectId}`

更新项目公共资料。字段与创建接口相同，并额外要求：

- `version`：详情接口返回的当前乐观锁版本。

不允许通过该接口修改可见性、审核状态和发布状态。

### `PUT /api/admin/projects/{projectId}/creators`

管理员全量设置项目参与创作者。请求字段：

- `version`：项目当前乐观锁版本。
- `creatorUserIds`：启用状态的创作者账号编号列表，不能重复。

如果项目由创作者创建，系统会自动保留该创建者。

## 工作台入口

- 正式地址：`/console/projects`。
- 管理员和创作者均可按权限查看、筛选、创建和编辑可访问项目。
- 管理员额外显示参与创作者分配入口；已发布项目的编辑和分配入口保持禁用。

## 当前边界

本阶段切片暂不提供项目删除、审核、发布、下架、可见性修改和封面上传接口。
分类标签和作品集协作接口见 `docs/阶段二作品集接口.md`，作品图片接口见
`docs/阶段二图片接口.md`。
