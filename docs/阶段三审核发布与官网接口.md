# 阶段三审核发布与官网接口

## 数据库版本与部署状态

Flyway `V6__add_collection_review_metadata.sql` 为 `work_collection` 新增：

- `rejection_reason`：作品集资料当前驳回原因。
- `submitted_at`：最近提交审核时间。
- `reviewed_at`、`reviewed_by`：最近审核时间和管理员。
- 审核队列索引 `review_status + submitted_at`。

Flyway `V6` 已于 2026-07-17 部署到生产环境。官网、工作台、公开分类、公开作品列表和
审核中心路由已完成正式域名验收；未认证审核接口返回 `401`，原图路径未公开并返回 `404`。

图片继续使用 `collection_photo` 现有审核字段。本阶段增加图片状态 `REJECTED`。

Flyway `V7__create_review_revision_tables.sql` 新增：

- `wedding_project` 的驳回原因、提交时间、审核时间和审核人字段。
- `review_task`：项目或作品集每次提交形成一条递增修订任务。
- `review_item`：保存字段或图片内容快照、审核状态、修订号和当前版本标记。
- `review_action_log`：保存提交、通过、驳回、撤回、移除等审核动作。
- 项目审核队列及审核任务、当前字段、当前业务图片的查询索引。

`V7` 已于 2026-07-17 部署到生产环境，生产 schema 当前为 `V7`。项目与作品集审核队列、
短期认证访问和工作台项目/作品集切换均已通过正式域名验收。

## 状态与并发规则

- 新作品集和新图片为 `DRAFT`。
- 新项目为 `DRAFT`；项目参与创作者可提交项目公共字段审核。
- 每次提交生成递增 `revisionNo`，字段和待审图片保存提交时快照。
- 首次提交生成全部字段审核项；后续只为内容发生变化的字段生成新审核项。
- 已通过且未修改的字段继续沿用原审核结论。
- 未修改的驳回字段继续保持驳回状态，不会被其他字段的新修订覆盖。
- 创作者提交作品集后，草稿或驳回图片变为 `PENDING`。
- 管理员可批量通过或驳回待审核图片，驳回必须填写原因。
- 管理员可多选项目或作品集字段分别通过或驳回，驳回必须填写原因。
- 当前字段或图片存在驳回时汇总为 `PARTIALLY_REJECTED`；无驳回但存在待审项时为 `PENDING`。
- 作品集封面有效、全部当前字段和未删除图片通过后进入 `APPROVED + READY`。
- 项目全部当前字段通过后进入 `APPROVED + READY`，项目发布接口尚未实现。
- 删除全部驳回图片后会重新计算状态；剩余图片全部通过且封面有效时自动进入 `READY`。
- 已通过图片不会因新图片重新提交而重复审核。
- 待审核期间修改项目或作品集内容，会撤回当前未处理审核项并回到 `DRAFT`；
  已有通过或驳回结论继续保留。
- 删除被驳回图片时，对应当前审核项标记为 `REMOVED`，物理文件继续保留。
- 所有修改、提交、审核、发布和下架操作继续校验作品集 `version`。
- 项目修改、提交和字段审核同样校验项目 `version`。
- `PUBLISHED` 状态锁定创作者编辑；管理员下架后恢复编辑。

## 创作者提交

以下接口要求权限资源 `/content/collections`，创作者还必须是作品集创建人或参与者。

### `POST /api/collections/{collectionId}/submit`

请求：

```json
{
  "version": 3
}
```

提交前必须至少存在一张未删除图片，并设置有效封面。

响应包含：

- `collection`：最新作品集详情和状态。
- `photoBatch`：最新作品集版本、封面和全部未删除图片审核状态。
- `reviewHistory`：当前审核项和按时间倒序的全部修订任务。

### `GET /api/collections/{collectionId}/review`

创作者或管理员查询当前作品集字段、图片审核项和完整修订记录。

## 项目提交与字段审核

项目创作者接口要求权限资源 `/content/projects`，创作者还必须是项目创建人或参与者。

### `POST /api/projects/{projectId}/submit`

请求包含当前 `version`。首次提交生成标题、新人展示名称、日期、地区编码、地点和介绍
六个字段审核项；再次提交只生成变化字段的新修订项。

### `GET /api/projects/{projectId}/review`

查询项目详情、当前字段审核项和历史修订记录。

以下管理员接口要求管理员角色和权限资源 `/review/collections`：

- `GET /api/admin/reviews/projects`
- `GET /api/admin/reviews/projects/{projectId}`
- `PUT /api/admin/reviews/projects/{projectId}/fields`
- `POST /api/admin/reviews/projects/{projectId}/approve`
- `POST /api/admin/reviews/projects/{projectId}/reject`

项目字段批量审核请求：

```json
{
  "version": 3,
  "reviewItemIds": [101, 102],
  "decision": "REJECT",
  "reason": "请调整公开展示名称"
}
```

`decision=REJECT` 时 `reason` 必填。项目全部当前字段通过后进入 `APPROVED + READY`。

## 管理员审核与发布

以下接口要求管理员角色和权限资源 `/review/collections`。

### `GET /api/admin/reviews/collections`

分页查询审核与发布工作列表。支持：

- `page`、`size`
- `keyword`
- `reviewStatus`
- `publishStatus`

### `GET /api/admin/reviews/collections/{collectionId}`

查询作品集资料、创作者、图片和当前审核发布状态。

### `PUT /api/admin/reviews/collections/{collectionId}/photos`

批量审核当前作品集中的待审核图片。

```json
{
  "version": 4,
  "photoIds": [11, 12],
  "decision": "APPROVE",
  "reason": null
}
```

`decision=REJECT` 时 `reason` 必填。

### `PUT /api/admin/reviews/collections/{collectionId}/fields`

按当前 `reviewItemIds` 批量通过或驳回作品集字段，请求结构与项目字段审核相同。
可审核关联项目、标题、介绍、主分类、标签和封面六个字段项。

### `POST /api/admin/reviews/collections/{collectionId}/approve`

当全部图片已通过时，一次通过所有剩余待审字段。不存在驳回项后进入 `READY`。

### `POST /api/admin/reviews/collections/{collectionId}/reject`

驳回作品集资料，请求包含 `version` 和必填 `reason`。

### `POST /api/admin/reviews/collections/{collectionId}/publish`

```json
{
  "version": 6,
  "visibility": "PUBLIC",
  "featured": true,
  "pinned": false,
  "sortOrder": 10
}
```

只有 `APPROVED + READY`、全部图片已通过且封面有效的作品集可以发布。
当前支持 `PUBLIC` 和 `HIDDEN`，`PASSWORD` 会明确返回暂不支持。

### `POST /api/admin/reviews/collections/{collectionId}/offline`

请求包含 `version` 和必填 `reason`。下架后公开接口立即不可访问，创作者可以继续编辑。

### `GET /api/admin/dashboard/overview`

待审核和部分驳回数量同时统计项目与作品集；可发布、已发布和最近审核列表继续以作品集为主。

## 官网公开接口

以下接口无需登录，只返回 `PUBLISHED + PUBLIC` 内容以及审核通过的水印预览图。

- `GET /api/public/content/categories`
- `GET /api/public/collections`
- `GET /api/public/collections/{collectionId}`

公开作品列表支持 `page`、`size`、`keyword` 和 `categoryId`。排序规则为置顶优先、
手动顺序优先、发布时间倒序。

公开响应不包含原图路径、文件摘要、创建账号或内部审核信息。已下架、隐藏和未发布作品
统一按公开内容不存在处理。
