## 接手顺序

1. 阅读根目录 `AGENTS.md`、本文和 `CHANGELOG.md`。
2. 执行 `git status --short`、`git log -5 --oneline`，确认分支及未提交改动。
3. 阅读本次要修改模块的代码、测试和接口文档，不根据交接摘要猜测实现。
4. 修改完成后执行对应最低验证，并同步更新本文与 `CHANGELOG.md`。
5. 只有在代码、测试、文档和必要的部署验证均完成后，才将任务标记为完成。

## 当前数据与安全约束

- 生产环境文件：`/etc/wedding-platform/wedding-platform.env`，权限 `0600`，所有者 `root`。
- 数据库密码和 JWT 密钥只保存在服务器环境文件中，不读取、不输出、不提交。
- 证书私钥不提交 Git；远端私钥权限为 `0600`。
- 原图目录不通过 Nginx 公开，后续只公开预览图和缩略图。
- 业务删除采用逻辑删除，图片物理文件长期保留。
- 生产引导必须保持 `BOOTSTRAP_ADMIN_ENABLED=false`。

## 已知注意事项

- iot 只有 JDK 17，不能将项目编译目标改回 Java 21。
- `www.photo.shop-hz.top` 尚非必需入口；若启用，需要域名所有者额外添加 A 记录。
- 当前首页和工作台仍是第一版展示，项目与作品集菜单尚未连接真实业务接口。
- 线上账号密码属于运行时凭据，不写入交接文档。
- 部署前保留用户未提交改动；不要使用破坏性 Git 命令。

## 常用命令

```bash
# 后端测试
cd wedding-server
./gradlew test

# 全量构建
./deploy/scripts/build-all.sh

# 仅发布应用
SKIP_BUILD=1 ./deploy/scripts/deploy-app.sh

# 证书与 Nginx 输入校验
DRY_RUN=1 ./deploy/scripts/deploy-nginx.sh

# 生产检查
ssh iot 'systemctl is-active wedding-platform.service'
ssh iot '/usr/local/nginx/sbin/nginx -t'
curl https://photo.shop-hz.top/api/public/status
```
