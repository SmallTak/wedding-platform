## 接手顺序

1. 阅读根目录 `AGENTS.md`、本文和 `CHANGELOG.md`。
2. 执行 `git status --short`、`git log -5 --oneline`，确认分支及未提交改动。
3. 阅读本次要修改模块的代码、测试和接口文档，不根据交接摘要猜测实现。
4. 修改完成后执行对应最低验证，并同步更新本文与 `CHANGELOG.md`。
5. 只有在代码、测试、文档和必要的部署验证均完成后，才将任务标记为完成。

## 已知注意事项

- iot 只有 JDK 17，不能将项目编译目标改回 Java 21。
- 当前开发机默认 `java` 为 11，运行 Gradle 时需要显式使用 `/opt/java/jdk17`。
- `www.photo.shop-hz.top` 尚非必需入口；若启用，需要域名所有者额外添加 A 记录。
- 工作台审核发布与官网作品展示已接入真实接口；客户反馈、数据统计和官网运营入口仍未实现。
- 生产审核模型已升级为字段与图片统一修订任务，历史 `V6` 作品集仍可继续查询和提交新修订。
- 密码访问、公开项目、逻辑删除和作品集列表下架已部署，尚待生产账号完整人工流程验收。
- 密码内容不进入公开列表，需要通过包含内容编号的直接链接访问。
- 项目存在未删除作品集时不能删除；已发布项目和作品集都必须先下架。
- 项目封面上传仍未实现，官网项目封面当前回退使用首个公开作品集封面。
- 常规部署不备份数据库；涉及破坏性 schema 变更时必须先改为兼容性迁移方案。
- 工作台生产构建仍提示主 chunk 超过 500 kB，后续页面接入时需要继续拆分。
- 线上账号密码属于运行时凭据，不写入交接文档。
- 部署前保留用户未提交改动；不要使用破坏性 Git 命令。
