# 代码复查记录（2026-07-21）

本文记录 2026-07-21 对阶段六代码的复查结论。经需求确认，本次官网视觉重设计只记录以下问题，
不修改后端数据、权限、访问控制、迁移或图片响应逻辑。

## P1

### 首页照片查询未限制公开可见性

- 位置：`CollectionPhotoRepository.findAllHeroPhotos()`、
  `HomepageService.heroPhotos()`。
- 现状：查询只校验作品集为已发布、审核通过，没有限制 `visibility = PUBLIC`；随后首页接口为
  每张照片生成受控高清图、预览图和缩略图地址。
- 影响：隐藏或密码作品集的图片地址可能进入公开首页响应。即使受控高清图接口还有二次访问校验，
  公开的预览图和缩略图静态地址仍存在泄露风险。
- 本次处理：首页前端已停止读取和渲染 `heroPhotos`，但后端仍会计算并返回该字段；隐私问题本次
  按要求不修复。

### 密码访问 Cookie 路径无法覆盖图片接口

- 位置：`PublicCollectionController.access()`、`PublicAccessCookies.write()`、
  `PublicOriginalImageController.image()`。
- 现状：解锁作品集后，Cookie Path 被设置为
  `/api/public/collections/{collectionId}`；作品高清图请求路径为
  `/api/public/images/photos/{photoId}/original`。
- 影响：浏览器不会把该 Cookie 发送给图片接口，密码作品集详情可解锁，但高清品牌图请求仍会因
  缺少访问令牌而失败。
- 本次处理：只记录，不修改 Cookie 范围或图片访问协议。

### V15 评价迁移存在任意归属和数据删除

- 位置：`V15__remove_wedding_projects.sql` 第 61 至 102 行。
- 现状：每条历史项目评价被关联到该项目最新的一条未删除作品集；找不到作品集时直接删除评价及
  回复。
- 影响：同一项目有多个作品集时评价可能被错误归属；没有可关联作品集的历史评价被不可逆删除。
- 本次处理：生产已执行 V15，本次只记录迁移风险，不补写迁移或改动生产数据。

## P2

### 首页 `heroPhotos` 无上限且存在 N+1 查询

- 位置：`CollectionPhotoRepository.findAllHeroPhotos()`、
  `HomepageService.heroPhotos()`。
- 现状：查询返回全部符合条件的照片；服务层再逐张调用 `assetRepository.findById()`。
- 影响：首页接口会随公开照片数量增长产生大量数据库查询和响应体；旧照片墙还会触发大量图片
  请求。
- 本次处理：首页已移除照片墙，不再触发这些图片请求；后端 N+1 和无界响应仍保留，未修复。

### 高清品牌图响应 MIME 可能与文件格式不一致

- 位置：`PublicOriginalImageController.image()` 与 `contentType()`。
- 现状：接口实际读取 `BrandedImagePathResolver` 生成的 JPEG 文件，但响应 Content-Type 仍根据
  原始资源的 MIME 判断；原图为 PNG 时会返回 `image/png`。
- 影响：在启用 `X-Content-Type-Options: nosniff` 的情况下，部分客户端或缓存层可能拒绝或错误
  处理该 JPEG 响应。
- 本次处理：只记录，不修改媒体响应。

### 创作者仍保留客户评价入口，但服务已要求管理员

- 位置：`/operations/feedback` 权限资源、工作台路由与导航、`FeedbackController`、
  `FeedbackService.requireAdmin()`。
- 现状：历史迁移把权限分配给管理员与创作者；前端继续按该权限展示入口和通知跳转，但列表、
  选项、新增、修改、撤回及回复等服务方法已统一要求管理员账号。
- 影响：有该权限的创作者仍能看到入口，但进入后请求会返回 `403`，形成权限模型与界面不一致。
- 本次处理：只记录，不修改角色权限、路由或服务授权。
