# 阶段二作品集接口

## 分类与标签配置

以下接口要求管理员角色和权限资源 `/config/content`：

- `GET /api/admin/content/categories`：查询全部未删除分类，包含停用项。
- `POST /api/admin/content/categories`：创建分类，初始状态为 `ACTIVE`。
- `PUT /api/admin/content/categories/{categoryId}`：编辑名称、介绍、排序和状态。
- `GET /api/admin/content/tags`：查询全部未删除标签，包含停用项。
- `POST /api/admin/content/tags`：创建标签，初始状态为 `ACTIVE`。
- `PUT /api/admin/content/tags/{tagId}`：编辑名称、排序和状态。

分类和标签名称忽略大小写校验重复。更新时必须提交当前 `version`。

拥有 `/content/collections` 权限的管理员和创作者可通过
`GET /api/collections/options` 查询启用状态的分类和标签，用于作品集表单。

## 权限与数据范围

作品集接口要求权限资源 `/content/collections`：

- 管理员可以查询和维护全部作品集。
- 创作者只能查询和维护自己创建或已被分配参与的作品集。
- 创作者创建作品集后自动成为共同创作者。
- 只有管理员可以分配作品集共同创作者。
- 关联婚礼项目时，创建或编辑作品集的创作者必须参与该项目。
- 关联项目作品集的所有共同创作者都必须是项目创建者或参与者。
- 更新内容、标签或共同创作者时必须提交最新 `version`。
- 已发布作品集保持锁定，必须在阶段三下架后才能修改。

## 作品集接口

### `POST /api/collections`

创建独立或关联项目的作品集。请求字段：

- `projectId`：可空，为空表示独立作品集。
- `title`：必填，最长 200 字符。
- `description`：可空，最长 5000 字符。
- `categoryId`：必填，必须是启用分类。
- `tagIds`：标签编号列表，最多 20 个，不能重复。

新作品集固定初始化为：

- `visibility=HIDDEN`
- `reviewStatus=DRAFT`
- `publishStatus=UNPUBLISHED`
- `sortOrder=0`
- `featured=false`
- `pinned=false`

### `GET /api/collections`

分页查询当前账号可访问的作品集。

查询参数：

- `page`：从 `0` 开始，默认 `0`。
- `size`：默认 `20`，范围 `1` 至 `100`。
- `keyword`：可选，匹配标题和介绍。
- `projectId`：可选，按婚礼项目筛选。
- `categoryId`：可选，按主分类筛选。

### `GET /api/collections/{collectionId}`

查询作品集详情、关联项目、分类、标签和共同创作者。

### `PUT /api/collections/{collectionId}`

更新作品集公共资料、关联项目、主分类和标签。字段与创建接口相同，并额外要求：

- `version`：详情接口返回的当前乐观锁版本。

已停用但原本已关联的分类或标签可以保留，不能新增选择。

### `PUT /api/admin/collections/{collectionId}/creators`

管理员全量设置作品集共同创作者。请求字段：

- `version`：作品集当前乐观锁版本。
- `creatorUserIds`：启用状态的创作者账号编号列表，不能重复。

创作者创建的作品集会自动保留创建者。关联项目作品集不能加入项目之外的创作者。

## 工作台入口

- 作品集管理：`/console/collections`。
- 分类标签管理：`/console/content-config`，仅管理员可见。
- 作品集表单支持独立作品集、关联项目、主分类和多标签维护。
- 管理员分配关联项目作品集共同创作者时，前端只列出该项目的启用参与者。

## 当前边界

本阶段切片暂不提供作品集删除、封面、图片上传、审核、发布、下架、可见性和运营排序接口。
封面与图片管理接口见 `docs/阶段二图片接口.md`。作品集删除、审核、发布、下架、
可见性和运营排序仍留在后续阶段。
